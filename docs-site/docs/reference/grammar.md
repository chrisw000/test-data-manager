# Grammar reference

Every example on this page is taken from the repository's executable
[`features/`](https://github.com/chrisw000/test-data-manager/tree/main/features) files,
which CI runs on every push — the documentation cannot drift from behaviour.

Entity names are matched **case- and plural-tolerantly** against the logical names shown by
`tdm list-entities` (`Customers` ≡ `Customer`). Property names match CLR properties
tolerantly too: `credit limit` ≡ `CreditLimit`. When you're unsure how a step will be
interpreted, ask the pipeline itself:

```bash
tdm explain 'an Order exists for Customer "Acme Ltd" with status "Pending"'
```

## Create

**Single entity**, with optional property overrides joined by `and`/commas:

```gherkin
Given a Customer exists with name "Acme Ltd" and tier "Gold" and credit limit "25000"
```

**Domain-qualified** (when two domains expose the same entity name):

```gherkin
Given a Billing Customer exists with name "X"
```

**References** — link to an earlier-created (or database-resolved) entity by natural key.
Repeatable for multiple principals:

```gherkin
Given an Order exists for Customer "Acme Ltd" with order number "ORD-1001" and status "Pending"
Given an Invoice exists for Account "A-1" for Customer "Acme Ltd" with amount "1250.00"
```

**Count bulk** — N generated rows (chunked `AddRange`, always via DbContext):

```gherkin
Given 500 Products exist with category "LoadTest"
```

**DataTable bulk** — one row per table line; a shared reference / shared defaults before the
colon apply to every row (per-row cells win):

```gherkin
Given the following Invoices exist for Account "Cleanup Account":
  | InvoiceNumber | Amount | Status | IssuedDate |
  | INV-D-01      | 10.00  | Draft  | today-3d   |
```

## Update

```gherkin
When the Customer "Acme Ltd" is updated with tier "Platinum"
```

## Delete

```gherkin
When the Product "TMP-0001" is deleted
When all Invoices with status "Draft" are deleted
When all Orders are deleted
```

## Verify (load)

Read + assert; also part of the benchmark surface:

```gherkin
Then an Order "ORD-1001" should exist with status "Pending"
Then 2 Products should exist with category "Widgets"
Then 0 Invoices should exist with status "Draft"
```

## External references (cross-domain)

Brings another domain's entity id into scope via the
[identity contract](decisions.md) — no cross-database transaction, no runtime coordination:

```gherkin
Given an external Customer reference "Acme Ltd" from Orders
```

The id is `UUIDv5("{owningDomain}|{Entity}|{naturalKey}")`, so both domains derive the same
GUID independently. Per-domain behaviour (`Synthesize`/`Verify`/`Trust`) and per-entity
projection seeding are configured in [settings](configuration.md#domains).

## Values

- Strings are quoted; numbers, bools, GUIDs, enums and dates convert automatically.
- Relative dates: `today`, `today-3d`, `today+2h` and similar offsets.
- Anything not overridden is generated by the domain's convention faker
  (`{Name}Faker`), or a heuristic auto-faker — deterministic under the scenario seed.

## Tags

| Tag | Where | Effect |
|---|---|---|
| `@seed:42` | feature/scenario | Pins the deterministic generation seed |
| `@domain:Billing` | feature/scenario | Pins entity resolution to one domain |
| `@benchmark` | feature/scenario | Forces benchmark stats for this scenario |
| `@skip` | scenario | Parsed but not executed (reported as skipped) |
| `@persistent` | scenario | Rows survive the run (overrides run lifecycle) |
| `@ephemeral` | scenario | Tracked teardown at scenario end |

`Background`, `Rule`, `Scenario Outline` + `Examples` (with `<placeholder>` substitution in
step text *and* DataTable cells) are fully supported.
